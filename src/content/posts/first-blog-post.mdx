This site is intentionally content-first. Every new post lives in `src/content/posts` as an `.mdx` file with frontmatter metadata.

## Why MDX

- I can write like normal Markdown.
- I still have the option to embed React components later.
- The content stays version controlled and easy to review.

## Suggested post structure

1. Problem and context
2. Constraints and decisions
3. Implementation notes
4. Lessons learned

As the project grows, this blog becomes a changelog of ideas, experiments, and outcomes.
